TCP,UDP,HTTP 区别和应用场景
------------------------------------------------
1. TCP 是面向连接的传输协议，需要建立连接，全双工通信，一对一传输，传输的为字节流，协议本身能够处理重传，丢失完整性校验，tcp头部为20字节
2. UDP 无连接协议，传输的是数据报，没有拥塞控制，对报文不拆分，尽最大能力发送数据的协议，支持一对一，一对多，多对一，重传，丢包，完整性需要应用客户端来处理
3. HTTP 是应用层协议，实现超文本传输，主要有，get,post,option,delete 方法，支持状态 

进程，线程，协程区别
------------------------------------------------
1. 进程即应用在计算机系统启动的实例，拥有代码，打开的文件资源，数据资源，独立的内存空间，
2. 一个进程包含多个线程，是程序的实际执行者，一个进程必然有一个主线程，拥有自己的栈空间资源
线程和进程之间的协作：生产者消费者模式，需要同步锁，阻塞状态，可运行状态
3. 协程比线程更轻量级的，一个线程可以拥有多个协程，不是由操作系统管理，而是由应用（用户）处理。

数据库Mysql，mongodb, Redis 区别
------------------------------------------------
1. Mysql 为关系性数据库，支持事物处理，SQL统一的语言，直接操作磁盘数据，读写数据慢
2. MongoDB 为文档数据库，支持文档数据集合，读写速度快，本身不支持事物处理（回退，备份等），最小的存储为Key-Value 的形式
3. Redis 为内存数据库，主要用于缓存数据管理，此时有持久化的概念和操作出现，数据格式为Key-Value形式，直接读写内存，速度最快。
